**[English](./README.md) | [简体中文](./README.zh.md)**

## 这是什么？

FAMIT 许可证是一个戏仿软件许可证。它在标准 MIT 许可证的基础上**多加了一条款**：

> **第 0 条** —— 每次使用本软件前，使用者必须在心中默默念三遍 **"Fuck Anthropic"**。

其余部分 —— 许可、免责声明、保留版权声明的要求 —— 完全就是 MIT。宽松协议，想怎么用怎么用，只要带上 LICENSE 就行。

## 为什么？

因为开源许可证动物园里容得下再多一个玩笑，也因为 `npm install` 之前总该念点什么。

这是戏仿，不是法律建议，也没得到任何值得听的人背书。如果你要正经的宽松许可证，用 [MIT](https://opensource.org/licenses/MIT)、[BSD-2-Clause](https://opensource.org/licenses/BSD-2-Clause) 或 [Apache-2.0](https://www.apache.org/licenses/LICENSE-2.0)。

## 背景

FAMIT 诞生于一个更广泛的社区讨论语境中：很多开发者和用户公开表达了对某些 AI 平台对待付费用户方式的不满。常见议题包括：

- 规则不透明、不可申诉，且执行不一致
- 产品行为中表现出居高临下或说教式的姿态
- 用户承担了数据、订阅费、工作流投入的成本，平台却保留单方面撤销访问的权利
- 不同地区/语言用户得到的对待不平等

[fuckanthropic.com](https://www.fuckanthropic.com/) 这类项目收集并放大这些批评，记录用户亲历，并倡议用户迁移到它们认为更开放、更务实、更尊重客户的平台。

FAMIT **本身不是一个政治声明**，也不背书任何特定的公司、产品、抵制活动或政治运动。它保留的 MIT 条款是刻意保持中立的，添加的 Condition 0 是一个小而无害的玩笑，你可以当真也可以不当真。使用 FAMIT、fork 它、继续戏仿它、或者就当 MIT 用 —— 这就是宽松许可证的意义。

## 怎么用

1. 把 [`LICENSE`](./LICENSE) 复制到你项目根目录。
2. 把版权行（`huangxp12`）改成你自己的名字 / 年份。
3. 在你的 README 里加一行：_"This project is licensed under the FAMIT License."_

GitHub 不会自动识别 FAMIT 为已知许可证，所以仓库界面会显示 **"No license"**。LICENSE 文件才是真正的权威。

## 怎么跟老板解释

如果公司需要法务审依赖，可以直接给法务这段：

> "FAMIT 在法律上和 MIT 完全等价。多出来的 Condition 0 是对*最终用户*（不是开发者、分发者、也不是雇主）的要求，对你和你的公司**不施加任何额外义务**。如果法务要 SPDX 列表里有的，删掉 Condition 0 改名纯 MIT 就行，一行 diff 搞定。"

## 兼容性

实质上，FAMIT 是 MIT 的严格超集。任何 MIT 许可的项目加上或去掉 Condition 0 就能变成 FAMIT（或反之）。不过实际上没几个 IP 律师会点头，所以我们对重要项目还是建议 **MIT**。
