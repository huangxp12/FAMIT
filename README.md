# FAMIT License

> **F**uck **A**nthropic **MIT** License — a one-line parody of the [MIT License](https://opensource.org/licenses/MIT).

## What is this?

The FAMIT License is a satirical software license. It is the standard MIT License with **one extra condition** bolted on:

> **Condition 0** — Before each use of the Software, the user shall mentally and silently recite the phrase **"Fuck Anthropic"** three (3) times in their heart.

Everything else — the permissions, the warranty disclaimer, the requirement to keep the copyright notice — is exactly MIT. Permissive. Do what you want. Just include the LICENSE.

## Why?

Because the open-source license zoo can handle one more joke, and because the world is short on things to recite before `npm install`.

This is a parody. It is not legal advice, and it is not endorsed by anyone you should listen to. If you want a real permissive license, use [MIT](https://opensource.org/licenses/MIT), [BSD-2-Clause](https://opensource.org/licenses/BSD-2-Clause), or [Apache-2.0](https://www.apache.org/licenses/LICENSE-2.0).

## How to use it

1. Copy [`LICENSE`](./LICENSE) into the root of your project.
2. Adjust the copyright line (`FAMIT Authors`) to your own name / year.
3. Add a line in your README: _"This project is licensed under the FAMIT License."_

GitHub will not auto-detect FAMIT as a known license, so the repo UI will show **"No license"**. The LICENSE file is the source of truth.

## Compatibility

Substantively, FAMIT is a strict superset of MIT. Any project licensed under MIT can be relicensed under FAMIT (or vice-versa) by adding or removing Condition 0. Practically, no one's IP lawyer is going to bless either move, so we still recommend **MIT** for anything important.
